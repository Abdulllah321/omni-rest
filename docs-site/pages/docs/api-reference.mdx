## Query Parameters

Omni Rest supports powerful filtering, sorting, and pagination via standard query parameters.

<Tabs items={['Pagination', 'Sorting', 'Filtering', 'Relation Filters', 'Search & Fields']}>
  <Tabs.Tab>
    ```javascript copy
    GET /api/product?limit=10&offset=20
    ```

    | Parameter | Type | Default | Description |
    |-----------|------|---------|-------------|
    | `limit` | number | 10 | Maximum number of records to return |
    | `offset` | number | 0 | Number of records to skip |

    #### Cursor-Based Pagination

    For larger datasets, cursor-based pagination is significantly faster (O(1)) than offset.

    ```javascript copy
    // 1. Initial request
    GET /api/product?limit=20&paginationMode=cursor

    // Response includes nextCursor
    // { "data": [...], "meta": { "hasMore": true, "nextCursor": "eyJpZCI6MjB9" } }

    // 2. Next page request
    GET /api/product?limit=20&paginationMode=cursor&cursor=eyJpZCI6MjB9
    ```

    - **O(1) Performance:** Fast queries regardless of depth.
    - **Stable results:** No skipped or duplicated items if records are inserted during pagination.
    - **Note:** In cursor mode, the response `meta` object omits `total` and `page`, replacing them with `nextCursor` and `hasMore`.
  </Tabs.Tab>
  <Tabs.Tab>
    #### Regular Sort

    ```javascript copy
    GET /api/product?sort=price:desc
    GET /api/product?sort=name:asc,createdAt:desc
    ```

    - Use `field:asc` for ascending (default when direction omitted)
    - Use `field:desc` for descending
    - Comma-separate multiple fields: `?sort=field1:asc,field2:desc`

    #### Sort by Relation Count

    Sort records by how many related records they have using the `_count.` prefix:

    ```javascript copy
    GET /api/department?sort=_count.employees:desc
    GET /api/category?sort=_count.products:asc
    ```

    Maps directly to Prisma's `orderBy: { relation: { _count: "desc" } }`.

    Compose with regular sort fields:

    ```javascript copy
    GET /api/category?sort=_count.products:desc,name:asc
    ```

    | Syntax | Description | Prisma equivalent |
    |--------|-------------|-------------------|
    | `?sort=name:asc` | Sort by field ascending | `orderBy: { name: "asc" }` |
    | `?sort=price:desc` | Sort by field descending | `orderBy: { price: "desc" }` |
    | `?sort=_count.posts:desc` | Sort by relation count descending | `orderBy: { posts: { _count: "desc" } }` |
    | `?sort=_count.tags:asc,name:asc` | Mixed count + field sort | `orderBy: { tags: { _count: "asc" }, name: "asc" }` |
  </Tabs.Tab>
  <Tabs.Tab>
    ```javascript copy
    GET /api/product?name=Laptop&price[gte]=100&price[lte]=1000&inventory[gt]=0
    ```

    #### Filter Operators

    | Operator | Description | Example |
    |----------|-------------|---------|
    | `eq` | Equal | `name=MacBook` |
    | `ne` | Not equal | `name[ne]=MacBook` |
    | `lt` | Less than | `price[lt]=1000` |
    | `lte` | Less than or equal | `price[lte]=1000` |
    | `gt` | Greater than | `price[gt]=100` |
    | `gte` | Greater than or equal | `price[gte]=100` |
    | `in` | In array | `categoryId[in]=1,2,3` |
    | `nin` | Not in array | `categoryId[nin]=1,2,3` |
    | `contains` | Contains substring | `name[contains]=Mac` |
    | `startsWith` | Starts with | `name[startsWith]=Mac` |
    | `endsWith` | Ends with | `name[endsWith]=Book` |
  </Tabs.Tab>
  <Tabs.Tab>
    #### Dot-Notation Relation Filtering

    Filter parent records by their related records' fields using `relation.field` syntax:

    ```javascript copy
    GET /api/department?employees.isActive=true
    GET /api/order?customer.city=Karachi
    GET /api/post?author.name_contains=john
    ```

    | Relation type | Syntax | Prisma equivalent |
    |---------------|--------|-------------------|
    | List (`isList: true`) | `?employees.isActive=true` | `{ employees: { some: { isActive: true } } }` |
    | Singular | `?customer.city=Karachi` | `{ customer: { city: "Karachi" } }` |
    | With operator | `?author.name_contains=john` | `{ author: { name: { contains: "john" } } }` |

    All filter operators (`_contains`, `_gte`, `_in`, etc.) work inside dot notation.

    Combines freely with regular filters:
    ```javascript copy
    GET /api/department?employees.isActive=true&name=Engineering
    ```

    <Callout type="info">
      Only one level of nesting is supported (`relation.field`). Deeply nested paths like `posts.comments.text` fall through to a regular filter.
    </Callout>
  </Tabs.Tab>
  <Tabs.Tab>
    #### Global Search

    ```javascript copy
    GET /api/department?search=engineering
    ```

    Performs a case-insensitive search across **all `String` fields** of the model simultaneously using an `OR` condition.

    ```
    → WHERE name LIKE '%engineering%'
         OR code LIKE '%engineering%'
         OR description LIKE '%engineering%'
    ```

    Combines with other filters:
    ```javascript copy
    GET /api/department?search=eng&isActive=true
    ```

    #### Field Selection

    Both `?select=` and `?fields=` are supported and produce identical results:

    ```javascript copy
    GET /api/user?select=id,name,email
    GET /api/user?fields=id,name,email
    ```

    When both are provided, `?select=` takes precedence.
  </Tabs.Tab>
</Tabs>

## Endpoints

### LIST - Get All Records

<Tabs items={['Request', 'Response (default)', 'Response (envelope: false)', 'Status Codes']}>
  <Tabs.Tab>
    **GET** `/api/:model`

    Returns a paginated list of records.

    **Query Parameters:**
    - `limit` (optional): Number of records per page
    - `offset` (optional): Number of records to skip (offset mode only)
    - `paginationMode` (optional): `offset` (default) or `cursor`
    - `cursor` (optional): Base64 encoded cursor for pagination
    - `sort` (optional): Sort field and direction, e.g. `name:asc` or `_count.posts:desc`
    - `search` (optional): Case-insensitive search across all String fields
    - `select` / `fields` (optional): Comma-separated field names to return
    - Filter parameters (optional): See filtering section
  </Tabs.Tab>
  <Tabs.Tab>
    ```json copy
    {
      "data": [
        {
          "id": 1,
          "name": "Laptop",
          "price": 999.99,
          "createdAt": "2024-01-01T00:00:00.000Z"
        }
      ],
      "meta": {
        "total": 42,
        "page": 1,
        "limit": 20,
        "totalPages": 3
      }
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    When `envelope: false` is set in options, returns a plain array:

    ```json copy
    [
      {
        "id": 1,
        "name": "Laptop",
        "price": 999.99
      }
    ]
    ```

    Total count is in the `X-Total-Count` response header.
  </Tabs.Tab>
  <Tabs.Tab>
    - `200` - Success
    - `400` - Bad request (invalid query parameters)
  </Tabs.Tab>
</Tabs>

### CREATE - Create New Record

<Tabs items={['Request', 'Response', 'Status Codes']}>
  <Tabs.Tab>
    **POST** `/api/:model`

    Creates a new record. Provide a JSON object with model fields (excluding auto-generated fields like `id`, `createdAt`).

    ```json copy
    {
      "name": "New Product",
      "price": 29.99,
      "inventory": 100
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```json copy
    {
      "id": 2,
      "name": "New Product",
      "price": 29.99,
      "inventory": 100,
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    - `201` - Created
    - `400` - Bad request (validation error)
    - `409` - Conflict (unique constraint violation)
  </Tabs.Tab>
</Tabs>

### READ - Get Single Record

<Tabs items={['Request', 'Response', 'Status Codes']}>
  <Tabs.Tab>
    **GET** `/api/:model/:id`

    Returns a single record by ID.

    **Path Parameters:**
    - `id` - Record ID
  </Tabs.Tab>
  <Tabs.Tab>
    ```json copy
    {
      "id": 1,
      "name": "Laptop",
      "price": 999.99,
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    - `200` - Success
    - `404` - Not found
  </Tabs.Tab>
</Tabs>

### UPDATE - Replace Record

<Tabs items={['Request', 'Response', 'Status Codes']}>
  <Tabs.Tab>
    **PUT** `/api/:model/:id`

    Replaces an entire record.

    **Path Parameters:**
    - `id` - Record ID

    **Request Body:**
    Complete JSON object with all required fields.
  </Tabs.Tab>
  <Tabs.Tab>
    Updated record object.
  </Tabs.Tab>
  <Tabs.Tab>
    - `200` - Success
    - `400` - Bad request
    - `404` - Not found
    - `409` - Conflict
  </Tabs.Tab>
</Tabs>

### PATCH - Update Record

<Tabs items={['Request', 'Response', 'Status Codes']}>
  <Tabs.Tab>
    **PATCH** `/api/:model/:id`

    Partially updates a record.

    **Path Parameters:**
    - `id` - Record ID

    **Request Body:**
    JSON object with only the fields to update.

    ```json copy
    {
      "price": 899.99,
      "inventory": 50
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    Updated record object.
  </Tabs.Tab>
  <Tabs.Tab>
    - `200` - Success
    - `400` - Bad request
    - `404` - Not found
  </Tabs.Tab>
</Tabs>

### DELETE - Delete Record

<Tabs items={['Request', 'Response (hard delete)', 'Response (soft delete)', 'Status Codes']}>
  <Tabs.Tab>
    **DELETE** `/api/:model/:id`

    Deletes a record by ID. If `softDelete: true` is configured and the model has a `deletedAt` or `isActive` field, the record is soft-deleted instead of destroyed.

    **Path Parameters:**
    - `id` - Record ID
  </Tabs.Tab>
  <Tabs.Tab>
    Hard delete — no response body, status `204`.
  </Tabs.Tab>
  <Tabs.Tab>
    Soft delete — returns the updated record with status `200`:

    ```json copy
    {
      "id": 1,
      "name": "Laptop",
      "deletedAt": "2026-03-31T17:00:00.000Z"
    }
    ```

    Or for `isActive` models:

    ```json copy
    {
      "id": 1,
      "name": "Laptop",
      "isActive": false
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    - `204` - Deleted (hard delete)
    - `200` - Updated (soft delete)
    - `404` - Not found
  </Tabs.Tab>
</Tabs>

### BULK UPDATE - Update Multiple Records

<Tabs items={['Request', 'Response', 'Status Codes']}>
  <Tabs.Tab>
    **PATCH** `/api/:model/bulk/update`

    Updates multiple records in one request.

    ```json copy
    {
      "ids": [1, 2, 3],
      "data": {
        "status": "active",
        "updatedAt": "2024-01-01T00:00:00.000Z"
      }
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```json copy
    {
      "updated": 3,
      "records": [
        { "id": 1, "status": "active" },
        { "id": 2, "status": "active" },
        { "id": 3, "status": "active" }
      ]
    }
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    - `200` - Success
    - `400` - Bad request
    - `404` - Some records not found
  </Tabs.Tab>
</Tabs>

<Tabs items={['Request', 'Response', 'Status Codes']}>

</Tabs>

## Error Responses

All endpoints return consistent error responses:

```json copy
{
  "error": {
    "message": "Record not found",
    "code": "NOT_FOUND",
    "details": {}
  }
}
```

## Authentication

If your application requires authentication, implement it as middleware before mounting the omni-rest adapter:

```typescript copy
app.use("/api", authenticateMiddleware, expressAdapter(prisma));
```
